Minis functions are quite simple.
They allow repetetive things to happen.
There are some modifiers that can also be used.

Here is the general function style:
```
fn <type> <name>() {}
```
This is the bare minimum. There are other things, like return modifiers.
It allows for even more static typing.
You can set things such as the onceset, which is like const, but it cannot be changed after being set, and can be determined at runtime.
Here is how an add function with a onceset type would look:
```
fn i128 add(i128 a i128 b) -> onceset {
  return a + b;
}
```

A macro is something that is guaranteed to be determined at compile time.
This is especially useful for targeting different systems.
The order the macros are searched for is defined as so...
1. #def
2. #ifdef
3. #if
4. #embed
The order was chosen specifically so as to minimize workload, and also via logic.

```
char array image = [
  #embed <img.png>
```

Minis has two ways of doing for loops. This is so to allow for complex or simple loops.
There is C++ style loops (complex looping), and Rust style loops (simple looping).

For simple looping, the bare minimum is start and end.
You can optionally add a variable to use within the loop, and a stepper, to determine how much to increase by.
If you don't define a variable, then you shouldn't include the "as"
The reason for an optional variable, is to minimize code bloat, and fulfill the idea of "Only use what you need."
```
for (<var> as <start>..<end>..<stepper>) {}
```
NOTE: Even if you don't define a variable for the loop, internally, there is still a variable. It just isn't accesable to you.
      The variable is used simply to keep track of the internal size.

Here are two simple loops. One prints "x" five times, the other prints the value of x:
```
// print "x"
for (1..5) {
  print("x");
}

// print value of x
for (x as 1..5) {
  print(x);
}
```

For complex looping, you define start, check, and stepper.
You must define all three. The compiler will not "guess" you want to increase by one.
You must also define the size of the number you will be using.

Here is a guide to complex looping:
```
for (<num type> <var> = <int>; <thing to check>; <stepper>) {}
```

Here is a complex loop to print the value of x:
```
for (u8 x = 0; x < 50; ++x) {
  print(x);
}
```

A while loop is used to do something until a value false.
These can also be used to make an infinite loop.
Here is one way to make an infinite loop, and exit upon "q" being pressed:
```
import keyboard as kb;
while (true) {
  if (kb::getKey() == "q") { break; }
}
```

Classes are simple. A class is used, to create simple objects, which inherit values.
They are comprised of funcitons and variables, seperated into public and private sectors.
A class cannot have a middleman, which is defined as anything not sitting within a private or public sector.

In a class, self refers to the current instance of itself.
It is only used within a class, and is especially useful for initialization.
It is also the only way to access the private sector.
Here is an example of using self, to print the color of the apple:
```
class apple {
  public {
    void display() {
      print("The apple is {}\n", self.color);
    }
  }
}
```

When making a class, you'll usually want to make an initiation function, which sets up variables.
The initiation lives in the public field.
The variables you set typically live in the private field.
Here is a simple initialization for something like an apple class:
```
apple(str color, float weight, tribool poisonus, array str atoms, bool discovered) {
  self.color = color;
  self.weight = weight;
  self.poisonus = poisonus;
  self.atoms = atoms;
  self.discovered = discovered;
}
```
